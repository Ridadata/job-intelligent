System Design

Platform Architecture

A clean monolithic architecture following the Medallion data pattern with strict 4-layer separation of concerns.

Layer 01

Data Ingestion

3 REST API clients (Adzuna, JSearch, France Travail) and 2 Scrapy spiders (Rekrute, Emploi.ma). All sources normalize output to a shared JobItem schema before Bronze ingestion.

Layer 02

ETL Processing

Airflow DAG runs every 6 hours. Bronze (immutable raw JSON) to Silver (Pydantic-validated) to Gold (enriched with MiniLM embeddings). Incremental via processed flag on Bronze rows.

Layer 03

Storage Layer

PostgreSQL 15 with pgvector extension (Supabase). HNSW index on vector(384) columns. Redis for hot recommendation cache (1h TTL). Row-level security on all tables.

Layer 04

Exposition Layer

FastAPI REST API with strict router to service to repository separation. React SPA consumes the API via a centralized api-client.ts. JWT authentication, TanStack Query client-side caching.

AI Engine

Matching & Recommendation System

A multi-signal composite scoring algorithm combining semantic similarity with structured business logic for transparent, explainable recommendations.

Composite Match Score

Weighted combination of four independent signals. Every result includes a full score breakdown for transparency.

score =
  0.50 × skill_overlap
  0.30 × embedding_cosine_similarity
  0.10 × seniority_alignment
  0.10 × location_match

Skill Overlap ×0.5
Highest Weight

Fuzzy matching between candidate skills and job requirements. Handles substring normalization and version-agnostic matching (e.g., “Python 3” matches “Python”).

Semantic Similarity ×0.3
pgvector cosine

Cosine distance via pgvector <=> operator on 384-dim Sentence-BERT embeddings. Captures semantic role alignment beyond keyword matching.

Seniority Alignment ×0.1
Experience fit

Compares candidate experience_years against job seniority_level (Junior / Mid / Senior). Penalises large mismatch in both directions.

Location Match ×0.1
Geographic

Substring matching between candidate preferred location and job location. Supports partial matches (e.g., “Paris” matches “Paris, Île-de-France”).

Security

Security Implementation

Production-grade controls across all platform layers, aligned with OWASP Top 10 mitigations.

JWT Authentication
HS256 signed tokens with 60-minute expiry. Issued on login, validated on every protected route via Depends(get_current_user). Token refresh mechanism planned for production.
python-jose
Password Hashing
bcrypt with 12 work-factor rounds via passlib. Passwords are never stored or logged in plain text. Timing-safe comparison used on verification to prevent timing attacks.
passlib · bcrypt
Row-Level Security
Supabase RLS enabled on all user-facing tables. Candidates can only read and write their own profiles, CVs, and saved jobs. Admins bypass via service-role key only.
Supabase RLS
Rate Limiting
Per-IP rate limiting applied globally via middleware. Thresholds configurable through environment variables. Returns 429 Too Many Requests with a Retry-After header.
SlowAPI Middleware
CORS Policy
Allowed origins restricted to an explicit allowlist configured via ALLOWED_ORIGINS environment variable. No wildcard * origin permitted in production environments.
FastAPI CORSMiddleware
Secrets Management
All configuration and secrets via environment variables. Pydantic BaseSettings reads from .env with validation at startup. No credentials in source code or version control.
pydantic-settings
